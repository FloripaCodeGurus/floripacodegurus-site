Django é um framework para desenvolvimento rápido para web, escrito em Python, que utiliza o padrão model-template-view (MTV). Foi criado originalmente como sistema para gerenciar um site jornalístico na cidade de Lawrence, no Kansas. Tornou-se um projeto de código aberto e foi publicado sob a licença BSD em 2005. O nome Django foi inspirado no músico de jazz Django Reinhardt.

Django é um framework de alto nível, escrito em Python que encoraja o desenvolvimento limpo de aplicações web.

Desenvolvido por experientes desenvolvedores, Django toma conta da parte pesada do desenvolvimento web, como tratamento de requisições, mapeamento objeto-relacional, preparação de respostas HTTP, para que, dessa forma, você gaste seu esforço com aquilo que realmente interessa: suas regras de negócio!

Quem está por trás disso?

O Django foi desenvolvido originalmente no World Online, o departamento WEB de um jornal em Lawrence, Kansas, USA. O Django agora é mantido por um time internacional de voluntários.

Quais sites usam Django?

DjangoSites.org possui uma lista crescente de sites construídos utilizando Django.

Mapeamento Objeto-Relacional (ORM)

Com o ORM do Django você define a modelagem de dados através de classes em Python. Com isso é possível gerar suas tabelas no banco de dados e manipulá-las sem necessidade de utilizar SQL (o que também é possível).

Interface Administrativa

No Django é possível gerar automaticamente uma interface para administração dos modelos criados através do ORM.

Sistema de Templates

O Django tem uma linguagem de templates poderosa, extensível e amigável. Com ela você pode separar design, conteúdo e código em Python.

O sistema de Cache do Django

A estrutura de cache do Django oferece oportunidades muito significativas para ganhos de desempenho, salvando o conteúdo dinâmico para que ele não precise ser calculado para cada solicitação.

Por conveniência, o Django oferece diferentes níveis de granularidade de cache: você pode armazenar em cache a saída de visualizações específicas, ou apenas as partes que são difíceis de produzir, ou até mesmo um site inteiro.

A implementação do armazenamento em cache não deve ser considerada como uma alternativa para melhorar o código que está com desempenho ruim porque foi gravado incorretamente. É um dos passos finais para produzir um código com bom desempenho, não um atalho.
